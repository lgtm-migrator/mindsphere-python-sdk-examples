TOKEN URLS

1-Switch token type from User token to Technical token and vice versa.

This end point will display after switching what is the current token type. All the subsequent api calls will use this new changed token type. If You want to make api call with other token type, then hit this end point again. http://127.0.0.1:8000/tokentype/

2-Current token.

You need to login to access the token. You can use this to view the current token.
http://127.0.0.1:8000/token

3-Check the logs You need to login to access the token.

http://127.0.0.1:8000/logs
http://127.0.0.1:8000/logs/?lines=100

4-Clear the logs

You need to have super user access

http://127.0.0.1:8000/cls

ASSET MANAGEMENT URLS

1. Create aspect type

input-Your tenant name http://127.0.0.1:8000/aspects/{Your_tenant}
save “id” and “name” from the response

2.Create asset type

input:Your tenant name, name From step1 response, id From step1 response
http://127.0.0.1:8000/assettype/{Your_tenant}?aspectname={nameFromstep1response}&aspectid={idFromstep1response}
save “id” from response

3.Create asset

input:“id” from step 2 response
http://127.0.0.1:8000/assets/{idFromStep2}/
save “etag” and “asset_id” from response

4.Create a file

input: Not reqd http://127.0.0.1:8000/assetfiles
save “id” from response

5.Associate the file with created asset

input:asset id from step3,etag from step3, any value for key,fileid from step 4
http://127.0.0.1:8000/assetfiles/{asset_id}?ifmatch={etag}&key={any_string}&fileid={fileid}
save “etag” from response

6.Add location to asset

input-asset id from step 3, asset etag from step 5
http://127.0.0.1:8000/assetlocation/{asset_id}/{asset_etag}

7.Get asset by ID

input-asset id from step 3 response
http://127.0.0.1:8000/assetsget/{idFromStep3}/
save “etag” Ensure the response has “file_assignments” and “location” populated

8.Delete asset by id

input-asset id from step 3 response, asset etag from step 7
http://127.0.0.1:8000/assetsdelete/{idFromStep3}?ifmatch={etag}

9.List all assets

input-not required
http://127.0.0.1:8000/assets

IOT FILESERVICE URLS

1. create file

input : path param (entity)
description : Create file
urls : https://mdspsdk-pythone2e-mdspsdk.eu1-int.mindsphere.io/fileservicecreate/{entity_id}
save : “file_path”

2. get file

input : path param (entity_id and file_path) and take path value from step1
description : Get content of the file
urls : https://mdspsdk-pythone2e-mdspsdk.eu1-int.mindsphere.io/fileservicegetfile/{entity_id}/{file_path}
output : get content of file_path

3. search file

input : entity id required
description : Under specified entity, list the number of files
urls : https://mdspsdk-pythone2e-mdspsdk.eu1-int.mindsphere.io/fileservicesearch/{entity_id}

4. multipart file : initialising the file

input : query param (upload = start) , path param (entity and file path) and path should be unique like ex : (india121)
description : For initialising the file, required to pass upload=start
urls : https://mdspsdk-pythone2e-mdspsdk.eu1-int.mindsphere.io/fileservicecreatemultipart/{entity_id}/{file_path}?upload=start
output : Intitated file upload for the path : filepath

4.1 multipart file : uploading the file part 1

input : query param (part_num = 1) and path param (entity and file path)
description : For uploading part requires minuium 5MB file and specify the part_num
urls : https://mdspsdk-pythone2e-mdspsdk.eu1-int.mindsphere.io/fileservicecreatemultipart/{entity_id}/{file_path}?part_num=1
output : Uploaded file for the part1

4.2 multipart file : uploading the file part 2

input : query param (part_num = 2) and path param (entity and file path)
description : For uploading part requires minuium 5MB file and specify the part_num
urls : https://mdspsdk-pythone2e-mdspsdk.eu1-int.mindsphere.io/fileservicecreatemultipart/{entity_id}/{file_path}?part_num=2
output : Uploaded file for the part2

4.3 multipart file : listing number of part uploaded properly

input : entity and path
description : Listed out number of parts uploaded sucessfully
urls : https://mdspsdk-pythone2e-mdspsdk.eu1-int.mindsphere.io/fileservicelistmultipart/{entity_id}/{file_path}
output : view number of the file part are uploaded sucessfully

4.4 multipart file : complete the file

input : query param (upload = complete) and path param (entity and file path)
description : After uploading the parts , pass query param (upload = complete)
urls : https://mdspsdk-pythone2e-mdspsdk.eu1-int.mindsphere.io/fileservicecreatemultipart/{entity_id}/{file_path}?upload=complete
output : Successfully uploaded file for the path : file path
save file path and provide file path in step 2 to see content of the file
