TOKEN URLS

1-Switch token type from User token to Technical token and vice versa.

This end point will display after switching what is the current token type. All the subsequent api calls will use this new changed token type. If You want to make api call with other token type, then hit this end point again. http://127.0.0.1:8000/tokenType/toggle

2-Current token.

You need to login to access the token. You can use this to view the current token.<br>

http://127.0.0.1:8000/token

3-Check the logs You need to login to access the token.

http://127.0.0.1:8000/logs<br>

http://127.0.0.1:8000/logs/?lines=100

4-Clear the logs

You need to have super user access

http://127.0.0.1:8000/cls

ASSET MANAGEMENT URLS

1. Create aspect type

input-Your tenant name http://127.0.0.1:8000/aspects?tenantName={tenant}<br>

save “id” and “name” from the response

2.Create asset type

input:Your tenant name, name From step1 response, id From step1 response<br>

http://127.0.0.1:8000/assets/assettype/{Your_tenant}?aspectname={nameFromstep1response}&aspectid={idFromstep1response}<br>

save “id” from response

3.Create asset

input:“id” from step 2 response<br>

http://127.0.0.1:8000/assets/assets/{idFromStep2}?parentid={parentid}<br>

save “etag” and “asset_id” from response

4.Create a file

input: Not reqd http://127.0.0.1:8000/assets/assetfiles<br>

save “id” from response

5.Associate the file with created asset

input:asset id from step3,etag from step3, any value for key,fileid from step 4<br>

http://127.0.0.1:8000/assets/assetfiles/{asset_id}?ifmatch={etag}&key={any_string}&fileid={fileid}<br>

save “etag” from response

6.Add location to asset

input-asset id from step 3, asset etag from step 5<br>

http://127.0.0.1:8000/assets/assetlocation/{asset_id}/{asset_etag}

7.Get asset by ID

input-asset id from step 3 response<br>

http://127.0.0.1:8000/assets/assetsget/{idFromStep3}<br>

save “etag” Ensure the response has “file_assignments” and “location” populated

8.Delete asset by id

input-asset id from step 3 response, asset etag from step 7<br>

http://127.0.0.1:8000/assets/assetsdelete/{idFromStep3}?ifmatch={etag}

9.List all assets

input-not required<br>

http://127.0.0.1:8000/assets/assets
